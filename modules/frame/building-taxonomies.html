Six Steps to Your Taxonomy

1. Define Domain

What phenomenon are you analyzing?

Example: "Russian strategic behavior in Ukraine"

2. Identify Dimensions

What are the independent ways to view it?

Test: Can they vary independently?

3. Develop Hierarchies

Break each dimension into L2 → L3 → Taxa

Maintain consistent abstraction levels

4. Test Orthogonality

Can something be high on A and low on B?

If not, merge or redefine

5. Validate Coverage

Does every instance fit somewhere?

Add "Mixed" or "Other" if needed

6. Operationalize

How will you classify real data?

Create classification guidelines

Prompting LLMs for Taxonomy Generation

taxonomy_prompt.txt
You are a domain expert creating a multi-dimensional taxonomy. DOMAIN: [Your research topic] Create a taxonomy with: - 5-9 orthogonal dimensions (HLTPs) - Each HLTP has 4 L2 elements - Each L2 has 3 L3 elements - Each L3 has 8 specific taxa Output format: TSV with columns HLTP | 2nd Level TE | 3rd Level TE | Taxon Ensure: ✓ Dimensional independence ✓ Complete coverage ✓ Observable criteria ✓ No ambiguous terms ✓ Consistent abstraction levels

Avoid These Traps

❌ Don't Do This

  • • Overlapping dimensions
  • • "Aggressive" in multiple places
  • • Mixing abstraction levels
  • • Ambiguous terms
  • • False orthogonality

✅ Do This Instead

  • • True independence
  • • Unique vocabulary per dimension
  • • Consistent levels
  • • Observable criteria
  • • Test with edge cases

Remember: If you can't vary one dimension while holding others constant,
they're not truly orthogonal
